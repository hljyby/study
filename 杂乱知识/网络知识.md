```shell
路由表
程序查路由表
1. 与子网掩码进行与操作 看与网络目标是否相符
例 192.168.0.104 他会与第10行 路由匹配
2. 获得匹配路由后通过接口向外发出请求，目标地址是网关地址
192.168.0.104 -> 192.168.0.104
# 上述 “在链路上” 的意思是网关地址和接口地址是一样的，可以不用通过路由器进行转发
3. 如果请求地址在路由表上没有找到，就进入到 0.0.0.0 里面 走默认网关也就是从
192.168.0.104 -> 192.168.0.1

# 接口 -> 我的理解每一个接口就是一个网卡（虚拟）
# 网关 -> 网络通信进行下一跳的地址 一般是路由器的地址 如 192.168.0.1
# 跃点数 -> 类似ping 的存活时间 每走一个借口新增一个，越小访问速度越快
# 网络目标 -> 请求ip与子网掩码进行与操作得到的10进制数与谁匹配就进入那个路由的凭证
```
